# ADR-0006: Strongly Typed IDs

## Status
Accepted

## Context
En sistemas con muchos IDs, Guid/string generan bugs de wiring.

## Decision
Usamos tipos específicos (PaymentId, etc.) para IDs de aggregates.

## Consequences
+ Seguridad de tipos.
+ Mejor legibilidad del modelo.
- Un poco más de fricción en mappers/EF.
