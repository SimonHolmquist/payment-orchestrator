# ADR-0001: Clean Architecture + Domain-Driven Design

## Status
Accepted

## Context
El sistema debe orquestar pagos y escalar: múltiples integraciones, procesos asíncronos (outbox/reconciliación) y alta testabilidad.

## Decision
Adoptamos una arquitectura en capas:
- Domain: reglas de negocio puras (aggregates, VOs, domain events).
- Application: casos de uso y orquestación (CQRS/MediatR cuando aplique).
- Infrastructure: EF Core, broker, repositorios, integraciones.
- Api: REST + concerns de transporte.
- Workers: procesamiento background (outbox publisher, reconciliación).

## Consequences
+ Test unitarios rápidos y confiables (Domain sin I/O).
+ Menos acoplamiento al stack de persistencia/mensajería.
- Más boilerplate y convenciones a respetar.
