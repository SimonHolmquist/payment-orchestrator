# ADR-0005: Money Value Object

## Status
Accepted

## Context
Montos y moneda son críticos: errores por usar decimal “suelto” son comunes.

## Decision
Representamos dinero con un Value Object (Money) con invariantes: monto no negativo (según caso), moneda, operaciones controladas.

## Consequences
+ Menos bugs de cálculo/moneda.
+ Expresa intención del dominio.
- Requiere mapeo explícito en persistencia/serialización.
